# Print Samples

Use this folder for real examples of what the legacy system prints today or what the replacement app should print in the future.

## What To Add

- scanned printouts
- PDFs
- screenshots
- photos of filled forms
- notes about paper size, margins, envelopes, labels, or letterhead

## Suggested Naming

- `notice_individual_current.pdf`
- `notice_batch_current.pdf`
- `payment_receipt_current.pdf`
- `deposit_summary_current.pdf`
- `monthly_financial_current.pdf`
- `mailing_labels_current.pdf`
- `id_card_current.pdf`

## Notes To Include

For each sample, try to note:

- what user action in dBase creates it
- whether it is single-item or batch
- whether it uses preprinted paper or letterhead
- whether page breaks matter
- whether window envelopes or label sheets are involved
- what fields must stay in the same exact place

## Tracking

Use `print_samples/print_inventory.csv` to track each print item and what it maps to in the legacy system.
